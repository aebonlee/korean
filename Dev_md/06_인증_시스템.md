# Korean Pro - 인증 시스템

## Supabase 설정

### 환경 변수 (`.env`)

```
VITE_SUPABASE_URL=https://hcmgdztsgjvzcyxyayaj.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhb...
```

### 클라이언트 초기화 (`src/lib/supabase.js`)

```js
const sbUrl = import.meta.env.VITE_SUPABASE_URL;
const sbAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY;
export const supabase = (sbUrl && sbAnonKey) ? createClient(sbUrl, sbAnonKey) : null;
```

- 변수 접두사: `sb` (Supabase prefix 패턴)
- Supabase가 미설정이면 `null` 반환 (개발 환경 대응)

## OAuth 인증 방식

### 1. Google OAuth

```js
const loginWithGoogle = async () => {
  const { error } = await supabase.auth.signInWithOAuth({
    provider: 'google',
    options: { redirectTo: window.location.origin }
  });
};
```

### 2. Kakao OAuth

```js
const loginWithKakao = async () => {
  const { error } = await supabase.auth.signInWithOAuth({
    provider: 'kakao',
    options: { redirectTo: window.location.origin }
  });
};
```

### Supabase 대시보드 설정 필요사항

1. **Google OAuth 설정**:
   - Google Cloud Console에서 OAuth 클라이언트 ID 생성
   - Supabase Dashboard > Authentication > Providers > Google 활성화
   - Client ID/Secret 입력

2. **Kakao OAuth 설정**:
   - Kakao Developers에서 앱 등록
   - Supabase Dashboard > Authentication > Providers > Kakao 활성화
   - REST API Key / Client Secret 입력
   - Redirect URI: `https://hcmgdztsgjvzcyxyayaj.supabase.co/auth/v1/callback`

## AuthContext 상태

```js
const { user, loading, loginWithGoogle, loginWithKakao, logout } = useAuth();
```

| 속성 | 타입 | 설명 |
|------|------|------|
| user | Object/null | 로그인된 사용자 정보 |
| loading | boolean | 인증 상태 로딩 중 |
| loginWithGoogle | function | Google OAuth 로그인 |
| loginWithKakao | function | Kakao OAuth 로그인 |
| logout | function | 로그아웃 |

## 보호된 라우트

`ProtectedRoute` 컴포넌트로 인증이 필요한 페이지를 보호합니다:

```jsx
<ProtectedRoute>
  <Dashboard />
</ProtectedRoute>
```

- 미인증 시 `/login`으로 리다이렉트
- 로딩 중에는 스피너 표시
