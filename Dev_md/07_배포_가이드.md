# Korean Pro - 배포 가이드

## 배포 환경

- **호스팅**: GitHub Pages
- **도메인**: korean.dreamitbiz.com
- **저장소**: https://github.com/aebonlee/korean
- **배포 도구**: gh-pages npm 패키지

## 배포 전 체크리스트

1. `.env` 파일에 Supabase 설정 확인
2. `CNAME` 파일에 도메인 확인 (`korean.dreamitbiz.com`)
3. `index.html` OG 메타태그 확인
4. OG 이미지 생성 완료 (`public/og-image.png`)

## 배포 명령어

### 1. 의존성 설치

```bash
npm install
```

### 2. OG 이미지 생성

```bash
npm run generate-og
```

이 명령은 `generate-og-image.js`를 실행하여 `public/og-image.png`를 생성합니다.
- 크기: 1200x630 (OG 표준)
- 디자인: 다크 블루 그라디언트 + "Korean Pro" 텍스트

### 3. 개발 서버

```bash
npm run dev
```

### 4. 프로덕션 빌드

```bash
npm run build
```

빌드 후 자동으로:
- `dist/` 디렉토리에 빌드 결과물 생성
- `dist/404.html` 생성 (SPA 라우팅 지원)
- `CNAME` 파일 복사

### 5. GitHub Pages 배포

```bash
npm run deploy
```

이 명령은:
1. `npm run build` 실행 (predeploy)
2. `gh-pages -d dist` 실행 (dist 폴더를 gh-pages 브랜치에 푸시)

## Vite 빌드 설정 (`vite.config.js`)

```js
export default defineConfig({
  plugins: [
    react(),
    {
      name: 'copy-404',
      closeBundle() {
        // dist/index.html → dist/404.html 복사 (SPA 라우팅)
        // CNAME 파일 복사
      }
    }
  ]
});
```

## GitHub Pages SPA 라우팅

GitHub Pages는 기본적으로 SPA 라우팅을 지원하지 않으므로:
1. `404.html`을 `index.html` 복사본으로 생성
2. 404 에러 시 React Router가 라우팅 처리

## DNS 설정

GitHub Pages에서 커스텀 도메인 사용:
1. GitHub 저장소 Settings > Pages > Custom domain에 `korean.dreamitbiz.com` 입력
2. DNS에 CNAME 레코드 추가: `korean.dreamitbiz.com` → `aebonlee.github.io`
